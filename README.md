# LLM-Wiki: 
Шаблон и архитектура для ведения базы знаний в Obsidian, где ИИ выступает в роли редактора и писателя, а не просто поисковика.

Этот подход — синтез идей трёх авторов: базовая концепция LLM-вики в обычном Markdown принадлежит Андрею Карпатому (Andrej Karpathy). Смысловая структура директорий заимствована у Farzaa. Правило «Lineage» (обязательная привязка всех фактов к исходникам в папке `raw/`) взято из методологии Брэда Грукса (Brad Groux).

Никаких сложных RAG-пайплайнов, векторных баз данных или эмбеддингов. Вся вики — это набор обычных текстовых файлов. Современные модели с огромным окном контекста (Gemini 1.5/3.1 Pro, Claude 3.5/4.6 Sonnet) легко переваривают миллионы токенов за раз. Проще и надежнее скармливать им весь текст целиком.

## Как это работает физически

1. **Сброс (Capture):** Пользователь кидает любой файл (заметку, лог, фото доски, голосовое сообщение) в папку `raw/`.
2. **Синтез (Synthesis):** ИИ-агент (например, OpenClaw, локальный скрипт или Claude Code) читает `raw/`, просматривает текущую структуру вики и органично обновляет нужные статьи. 
3. **Результат (Result):** Агент делает git commit, и в Obsidian появляются готовые, переписанные начисто страницы со ссылками на оригиналы.

## Варианты синхронизации (Sync Topology)

Так как в системе участвуют мобильные устройства, ПК и ИИ-агент (сервер/облако), важно правильно настроить потоки данных:

**Вариант А: Только Git (Бесплатный)**
Все устройства и ИИ-агент синхронизируются через единый Git-репозиторий.
*   **Плюсы:** Полностью бесплатно. Единый источник истины.
*   **Минусы:** Git на мобильных устройствах (особенно iOS) работает через сторонние приложения (например, Working Copy) и может быть неудобным. Есть риск merge-конфликтов, если ИИ и пользователь одновременно редактируют файлы.

**Вариант Б: Гибридный (Obsidian Sync + Git-мост)**
Мобильные устройства и ПК синхронизируются через нативный Obsidian Sync. ПК дополнительно настроен на авто-пуш в приватный Git-репозиторий (например, через плагин Obsidian Git), откуда файлы забирает ИИ-агент.
*   **Плюсы:** Бесшовный опыт на смартфоне. ИИ работает в изолированном контуре (Git) и не ломает синхронизацию на телефоне.
*   **Минусы:** Требуется платная подписка Obsidian Sync.

## Структура папок (Digital Garden)

Каталогизация здесь идет не по типу данных, а по смыслу. Рекомендуемая базовая структура для личного дневника/вики:

- `raw/` — сырые исходники (только чтение, ИИ их никогда не меняет).
- `wiki/philosophies/` — принципы работы и жизни.
- `wiki/patterns/` — повторяющиеся поведенческие циклы.
- `wiki/decisions/` — важные решения (почему было выбрано именно это).
- `wiki/tensions/` — неразрешимые конфликты ценностей или приоритетов.
- `wiki/eras/` — эпохи жизни.
- `wiki/projects/` — проекты (от задумки до завершения).
- `wiki/people/` — люди и их влияние.

## Три главных правила для ИИ (AGENTS.md)

Системный промпт для модели (см. `AGENTS.md`) держится на трех китах:
1. **Никакой отсебятины.** Статьи пишутся сухо, как в Википедии. Эмоции передаются только через прямые цитаты из сырых заметок.
2. **Происхождение (Lineage).** Любой добавленный факт должен содержать `[[ссылку]]` на исходный файл в `raw/`. Пользователь всегда должен иметь возможность кликнуть и проверить, откуда ИИ взял информацию.
3. **Писатель, а не клерк.** ИИ не должен просто дописывать новые события в конец файла (как хронологический лог). Он должен органично вплетать новые данные в существующий текст, чтобы статья читалась как единое целое.
