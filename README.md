# LLM-Wiki

A template for maintaining a personal knowledge base in Obsidian, where AI acts as an editor and writer — not just a search engine.

This approach synthesizes ideas from three authors: the core concept of an LLM-wiki in plain Markdown belongs to **Andrej Karpathy**. The semantic directory structure is borrowed from **Farzaa**. The "Lineage" rule (mandatory linking of all facts to source files in `raw/`) is adopted from **Brad Groux's** methodology.

No complex RAG pipelines, vector databases, or embeddings. The entire wiki is a set of plain text files. Modern models with massive context windows can ingest millions of tokens at once — it is simpler and more reliable to feed them the full text.

---

## How It Works

1. **Capture:** Drop any file (a quick note, a log, a whiteboard photo, a voice memo) into the `raw/` folder.
2. **Synthesis:** An AI agent (Claude Code, OpenClaw, or a local script) reads `raw/`, reviews the current wiki structure, and organically updates or creates relevant articles.
3. **Result:** The agent commits the changes via `git commit`, and polished, fully rewritten pages with links to the original sources appear in your Obsidian vault.

## Directory Structure

```
llm-wiki/
├── raw/                <- Raw sources (read-only, AI never modifies these)
│   └── entries/        <- One .md per entry (generated by ingest)
├── wiki/               <- Compiled knowledge (AI-managed)
│   ├── projects/       <- Projects (from idea to outcome)
│   ├── philosophies/   <- Work and life principles
│   ├── patterns/       <- Behavioral patterns (trigger -> reaction -> outcome)
│   ├── decisions/      <- Important decisions (context and reasoning)
│   ├── transitions/    <- Transitional periods
│   ├── eras/           <- Life epochs
│   ├── people/         <- People and their impact
│   ├── tensions/       <- Unresolvable value conflicts or trade-offs
│   ├── concepts/       <- Other semantic nodes
│   ├── _index.md       <- Master index with aliases
│   └── _backlinks.json <- Reverse link index
├── AGENTS.md           <- AI agent instructions
└── ARCHITECTURE.md     <- System architecture and design rationale
```

## Three Core Rules for the AI

The system prompt (see `AGENTS.md`) rests on three pillars:

1. **No editorializing.** Articles are written in a dry, encyclopedic tone. Emotions are conveyed *only* through direct quotes from the raw notes.
2. **Lineage.** Every added fact must contain a `[[wikilink]]` to its source file in `raw/`. You should always be able to click through and verify where the AI got the information.
3. **Writer, not a clerk.** The AI does not simply append new events to the end of a file. It organically weaves new data into the existing text so the article reads as a cohesive narrative.

## Getting Started

1. **Clone this template:**
   ```bash
   git clone https://github.com/indi256s/llm-wiki-template.git my-wiki
   cd my-wiki
   ```

2. **Open the folder in Obsidian** as a vault.

3. **Drop your first raw file** into `raw/` (a text note, journal entry, etc.).

4. **Run the AI agent** to process it:
   ```
   /wiki ingest
   /wiki absorb all
   ```

5. **Browse your wiki** in Obsidian — articles appear in `wiki/` with full wikilinks and source tracing.

## Sync Setup

Since personal knowledge management involves mobile devices, PCs, and an AI agent, choosing the right sync topology is critical. Two options are supported:

### Option A: Pure Git (Free)

All devices and the AI agent sync via a single Git repository. Best for users comfortable with Git on all devices.

**Setup:**

1. Create a **private** GitHub/GitLab repository and push this template to it.

2. **Desktop (Mac/Windows/Linux):**
   - Open the vault in Obsidian.
   - Install the [Obsidian Git](https://github.com/denolehov/obsidian-git) community plugin.
   - Configure it:
     - **Auto pull interval:** `5` minutes (or your preference).
     - **Auto push after commit:** enabled.
     - **Pull on startup:** enabled.
   - The plugin will auto-commit and push/pull changes on a schedule.

3. **Mobile (iOS):**
   - Install [Working Copy](https://workingcopyapp.com/) (Git client for iOS).
   - Clone your repository in Working Copy.
   - In Obsidian mobile, open the cloned folder as a vault.
   - To sync: open Working Copy, pull changes, edit in Obsidian, then commit and push in Working Copy.
   - Optionally, set up iOS Shortcuts to automate pull/push on app open/close.

4. **Mobile (Android):**
   - Install [Termux](https://termux.dev/) and set up Git inside it.
   - Clone the repo to shared storage or use [MGit](https://github.com/maks/MGit).
   - Open the cloned folder in Obsidian mobile as a vault.
   - Pull/push manually via Termux or MGit.

5. **AI Agent (Server/Cloud):**
   - The agent clones the same repo, runs on a schedule (cron) or on demand.
   - It pulls, processes `raw/`, writes to `wiki/`, commits, and pushes.

**Pros:** Completely free. Single source of truth. Full version history.
**Cons:** Git on mobile can be clunky. Risk of merge conflicts if the AI and user edit simultaneously.

**Conflict prevention tips:**
- The AI only writes to `wiki/`. Users only write to `raw/`. This separation eliminates most conflicts.
- Set the AI agent to pull immediately before processing and push immediately after.

---

### Option B: Obsidian Sync + Headless Client

All devices sync via native Obsidian Sync. The AI agent uses [Obsidian Headless](https://obsidian.md/help/sync/headless) — an official CLI client that connects directly to Obsidian Sync without the desktop app. No Git involved.

**Setup:**

1. **Subscribe to [Obsidian Sync](https://obsidian.md/sync)** (paid plan).

2. **Mobile + Desktop:**
   - Open the vault on all devices.
   - Enable Obsidian Sync in Settings > Core plugins > Sync.
   - Connect all devices to the same remote vault.
   - Edits on phone appear on desktop instantly, and vice versa.

3. **AI Agent (Server/Cloud):**
   - Install the headless client:
     ```bash
     npm install -g obsidian-headless
     ```
   - Log in and connect to the vault:
     ```bash
     ob login
     ob sync-setup --vault "My Vault" --path ~/my-wiki
     ```
   - Configure for the agent's use case — the agent only needs to read `raw/` and write to `wiki/`:
     ```bash
     ob sync-config --mode bidirectional
     ```
   - Run a one-time sync before and after processing:
     ```bash
     ob sync                    # pull latest
     # ... agent processes raw/entries/, writes to wiki/ ...
     ob sync                    # push changes
     ```
   - Or run continuous sync in the background:
     ```bash
     ob sync --continuous
     ```

**Data flow:**
```
Phone  <--Obsidian Sync-->  Desktop
                  \
                   `--Obsidian Sync-->  AI Agent (obsidian-headless)
```

**Pros:** Seamless experience on all devices. No Git setup needed anywhere. End-to-end encryption. The AI agent is just another Obsidian Sync device.
**Cons:** Requires a paid Obsidian Sync subscription (~$4/month billed annually).

## Using the `/wiki` Skill (Claude Code)

This template includes a Claude Code skill at `.claude/skills/wiki/SKILL.md`. If you use Claude Code as your AI agent, you get these commands:

| Command | Description |
|---------|-------------|
| `/wiki ingest` | Convert source data files into individual markdown entries in `raw/entries/` |
| `/wiki absorb [range]` | Compile raw entries into wiki articles (default: last 30 days) |
| `/wiki query <question>` | Ask questions about the wiki content |
| `/wiki cleanup` | Audit and enrich existing articles |
| `/wiki breakdown` | Find and create missing articles |
| `/wiki status` | Show wiki statistics |
